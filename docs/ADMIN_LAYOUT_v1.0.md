## 🎼 Sugar Rush — 발매 프로세스 중심 순서형 흐름도

```
[1. 제작 시작]
   └─ 제작·협업 보드
        - 단계: 작사중 → 앨범아트 준비중 → 작곡대기중 → 교정중
        - 각 단계별 담당자 기록(@멘션, 피드백)

      ↓

[2. 업로드 & 검증]
   └─ 업로드·검증 센터
        - 이미지 업로드(1:1, 3000px 이상, 컬러 추출/대체안)
        - 오디오 업로드(WAV 조건 검증, HLS 프리뷰)
        - 권리 검증(분배율 합=100% + 증빙 첨부)
        - 링크 검증(정규식+HEAD, 플랫폼 화이트리스트)

      ↓

[3. 앨범/트랙 관리]
   ├─ 앨범 관리
   │    - 앨범 기본정보 입력(이름, 소개, 발매일, 표지)
   │    - 아티스트 사진/소개(전역 적용)
   │
   └─ 트랙 관리
        - 트랙명(한/영), 가사(Yjs 편집), WAV 등록
        - 원곡/참고곡, 버전 이력 기록

      ↓

[4. 발매 준비]
   └─ 발매 관리
        - 발매 예약(날짜, 플랫폼 선택)
        - 링크체커 실행(Apple→YouTube→Melon→Flo→Vibe)
        - 실패 시: 백오프 재시도 / 수동 확정
        - 발매 상태 모니터링(예정 → 진행중)

      ↓

[5. 일정 반영]
   └─ 일정·캘린더
        - 발매일 자동 등록
        - 일정 수정 시 이력 보존
        - 알림(D-7/D-1/D-day 푸시)

      ↓

[6. 발매 완료 → 홈페이지 자동 노출]
   └─ 공개 사이트
        - 앨범 카드(썸네일, 테마 컬러, 발매일, 플랫폼 버튼)
        - 상세 페이지(히어로, 소개, 트랙 리스트, HLS 프리뷰)

      ↓

[7. 수익 정산]
   └─ 수익 관리
        - 정산 원천 데이터 임포트(API/CSV/엑셀 업로드)
        - 임계 평가 후 READY 카드 생성
        - 출금 요청 → 2인 승인 → 지급(PAID)
        - 지출 관리(Suno 구독료 등), PDF 증빙 다운로드
        - 통계: 일별/월별/사이트별/앨범별/곡별 그래프

      ↓

[8. 피드백 & 운영]
   ├─ 피드백·알림 센터
   │    - Google Chat 알림(Severity/⏳)
   │    - PWA 알림(중요만/요약/NR 15분 묶음)
   │    - CSAT/Thumbs/TAT 지표
   │
   └─ 보안·운영 관리
        - 금칙어 게이트(CI 차단/예외 승인)
        - 사용자·역할, 2FA/Argon2id
        - 감사 로그/백업·DR
```

## 📊 흐름 요약

**제작 시작**: 협업 보드(칸반 단계) →

**업로드/검증**: 이미지·오디오·권리·링크 체크 →

**앨범/트랙 관리**: 정보 입력·버전 관리 →

**발매 준비**: 예약·링크체커·상태 모니터링 →

**일정 반영**: 캘린더 자동 등록·알림 →

**발매 완료**: 홈페이지 자동 노출 →

**수익 정산**: 임포트→임계→출금·승인·지급 →

**피드백/운영**: 알림·CSAT 지표·보안/운영 관리
